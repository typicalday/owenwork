# Reaper/Commit Safety — Build Plan

## Goal

Fix one liveness bug: Engine.close() and Engine.reap() overwrite alarm_at with NULL when they call store.putTask() without an alarmAt field. This silently destroys any alarm the worker set via set-alarm immediately before calling close. Add regression tests that lock in (a) alarm-survives-close/reap behavior and (b) lease-ownership enforcement at commit time.

No public API shape changes. No new dependencies. npm run check (tsc --noEmit + full test suite on node 22 and 24) must pass.

## Files to Touch

1. src/engine.ts — patch the three putTask calls that omit alarmAt inside close(), the internal releaseLeaseOnBornReject(), and reap().
2. test/engine.test.ts — add the two new regression test blocks.

No other files need changes.

## Fix: src/engine.ts

### Root cause

Store.putTask() writes alarm_at = data.alarmAt ?? null via ON CONFLICT DO UPDATE SET alarm_at = excluded.alarm_at. Whenever any of the three release-lease paths call putTask without alarmAt, the field resolves to null and the upsert NULLs alarm_at in the DB, clobbering whatever setAlarm wrote moments before.

### Chosen approach: read-then-pass-through in close/reap

Each of the three sites already reads the task row before calling putTask. Propagate task.alarmAt when defined. Minimal, targeted; leaves claim() untouched.

### Patch 1 — Engine.close() lines ~878-881

Existing call:
  this.store.putTask({ workflow, loop: r.loop, key: r.key ?? '', status: 'idle', attempts: task.attempts });

Replace with:
  this.store.putTask({
    workflow,
    loop: r.loop,
    key: r.key ?? '',
    status: 'idle',
    attempts: task.attempts,
    ...(task.alarmAt !== undefined ? { alarmAt: task.alarmAt } : {}),
  });

### Patch 2 — Engine.reap() lines ~899-905

Existing call:
  this.store.putTask({ workflow, loop: task.loop, key: task.key, status: 'idle', attempts: task.attempts + 1 });

Replace with:
  this.store.putTask({
    workflow,
    loop: task.loop,
    key: task.key,
    status: 'idle',
    attempts: task.attempts + 1,
    ...(task.alarmAt !== undefined ? { alarmAt: task.alarmAt } : {}),
  });

### Patch 3 — Engine.releaseLeaseOnBornReject() lines ~1189-1191

Existing call:
  this.store.putTask({ workflow, loop: r.loop, key: r.key ?? '', status: 'idle', attempts: task.attempts });

Replace with:
  this.store.putTask({
    workflow,
    loop: r.loop,
    key: r.key ?? '',
    status: 'idle',
    attempts: task.attempts,
    ...(task.alarmAt !== undefined ? { alarmAt: task.alarmAt } : {}),
  });

### Invariant to NOT break

claim() at lines ~523-531 calls putTask with { status: 'claimed', run: runId, claimedAt: now, attempts: ... } and no alarmAt. This intentionally NULLs alarm_at at claim time (the idle firing is consumed on claim). Leave it UNTOUCHED.

### Test block 1 — Alarm survives close and reap; claim clears it

test('alarm set before close is preserved; reap also preserves alarm; claim clears it', () => {
  const idleDef = def('alarm-survive', [input('proposal')], [
    loop({ name: 'planner', consumes: ['proposal'], produces: ['plan'] }),
    loop({ name: 'completion', produces: ['outcome'], on: ['idle'], idleAfterMs: 9_999_999 }),
  ]);
  const { engine, store } = makeEngine([idleDef], { reapTtlMs: 500 });
  const wf = engine.createInstance('alarm-survive');
  const ALARM = 99_999;

  // Part A: alarm survives a normal close
  const runId1 = 'run_close_test';
  store.insertRun(runId1, { workflow: wf, loop: 'completion', key: '' });
  store.putTask({ workflow: wf, loop: 'completion', key: '', status: 'claimed',
    run: runId1, claimedAt: 1000, attempts: 0, alarmAt: ALARM });
  engine.close(wf, runId1, 'ok');
  assert.equal(store.getAlarm(wf, 'completion'), ALARM, 'close() must not clear a freshly-set alarm');

  // Part B: alarm survives reap
  // (reapTtlMs=500; claimedAt=0; now=1000 => 1000-0=1000 > 500 => stale)
  const runId2 = 'run_reap_test';
  store.insertRun(runId2, { workflow: wf, loop: 'completion', key: '' });
  store.putTask({ workflow: wf, loop: 'completion', key: '', status: 'claimed',
    run: runId2, claimedAt: 0, attempts: 1, alarmAt: ALARM });
  engine.reap(wf, 1000);
  assert.equal(store.getAlarm(wf, 'completion'), ALARM, 'reap() must not clear a set alarm');

  // Part C: claim-time consume still works
  engine.setAlarm(wf, 'completion', 1); // past => immediately due
  const t = engine.tick(wf, { now: 2 });
  const completionOrder = t.orders.find((o) => o.loop === 'completion');
  assert.ok(completionOrder, 'completion must fire when idle alarm is due');
  assert.equal(store.getAlarm(wf, 'completion'), undefined,
    'claim() must clear alarm_at at claim time');
});

Note: store.insertRun is the public method on Store. makeEngine returns { engine, store }. The makeEngine helper already accepts { reapTtlMs } as its second arg.

### Test block 2 — Lease ownership at commit (openRun guard)

test('openRun: a reaped or superseded run cannot commit', () => {
  const { engine } = makeEngine([delivery], { reapTtlMs: 0 });
  const wf = engine.createInstance('delivery');

  // Claim planner as R1 at T=1000
  const t1 = engine.tick(wf, { now: 1000 });
  const r1 = t1.orders.find((o) => o.loop === 'planner');
  assert.ok(r1, 'planner must fire on first tick');

  // Reap at T=1001 (1001-1000=1 > ttl=0 => stale)
  engine.reap(wf, 1001);

  // Sub-case A: green on reaped run must throw
  assert.throws(
    () => engine.green(wf, r1.run, 'plan', { v: 1 }),
    /no longer holds its lease|reaped or superseded/,
    'green on a reaped run must throw'
  );

  // Sub-case B: R2 re-claims; R1 green must still throw
  const t2 = engine.tick(wf, { now: 2000 });
  const r2 = t2.orders.find((o) => o.loop === 'planner');
  assert.ok(r2, 'planner must re-fire after reap');
  assert.notEqual(r2.run, r1.run, 'R2 must be a distinct run id');
  assert.throws(
    () => engine.green(wf, r1.run, 'plan', { v: 1 }),
    /no longer holds its lease|reaped or superseded/,
    'green on superseded run must throw even after re-claim'
  );
});

Note: openRun() at line ~1264-1273 already throws the string 'run X no longer holds its lease (reaped or superseded)'. The regex /no longer holds its lease|reaped or superseded/ matches it. Do NOT change openRun's message to fit the test; adjust the regex if wording ever changes.

## Edge cases and constraints

- releaseLeaseOnBornReject runs inside an open tx begun by the caller; the getTask read and putTask write are coherent within that tx. No locking change needed.
- close() and reap() already hold the task object in a local variable that includes alarmAt from mapTask. No additional reads needed.
- alarmAt is optional on TaskData; conditionally spreading only when !== undefined matches the existing pattern (data.alarmAt ?? null in putTask). Idiomatic with the rest of the file.
- The three patches are structurally identical. Treat each as a separate targeted change; do not consolidate into a shared helper.
- Zero new imports, exports, or public API surface changes.
