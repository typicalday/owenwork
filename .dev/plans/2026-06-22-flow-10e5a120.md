# Plan: Add `.dev/checks.sh` and dogfood it in CI

## Goal

Commit a `.dev/checks.sh` contract to liveloop so the dev delivery checker can execute it (exit 0 → `checks: passed`; nonzero → `checks: failed`; missing → escalation stall). Wire CI to call the same script. This unblocks all future liveloop delivery instances from stalling at the checker step.

## Background / context

- `.gitignore` line 13 is a bare `.dev/` entry that git treats as a **directory exclusion** — git stops descending into `.dev/` entirely, so `!.dev/checks.sh` negation has no effect. The fix is `"ignore-all-but-one"`: replace `.dev/` with `.dev/*` + `!.dev/checks.sh`.
- `package.json` already defines `"check": "npm run typecheck && npm run test:quiet"` (`tsc --noEmit` + `node --test`). The script must delegate to this, never inline the commands.
- `.github/workflows/ci.yml` line 36 runs `npm run check` and carries a stale comment on line 33 referencing `better-sqlite3` (liveloop no longer depends on it).
- `.dev/checks.sh` does not exist yet.

## Step-by-step

### 1. Fix `.gitignore`

In `.gitignore`, replace the block at lines 12-13:

```
# dev delivery-system scaffolding (local foreman state — not part of the library)
.dev/
```

with:

```
# dev delivery-system scaffolding (local foreman state — not part of the library)
.dev/*
!.dev/checks.sh
```

Reason: `.dev/*` ignores the directory's contents but not the directory itself, so git descends in and the `!.dev/checks.sh` negation takes effect. This keeps `project.sh`, `plans/`, `workflows/`, etc. untracked.

**Verify immediately after editing:**
```bash
git check-ignore -v .dev/checks.sh   # must print nothing (not ignored)
git check-ignore -v .dev/project.sh  # must still print a match (.gitignore:13:...)
```

### 2. Create `.dev/checks.sh`

Create the file at `/Users/alexrojas/code/liveloop/main/.dev/checks.sh` with exact contents:

```bash
#!/usr/bin/env bash
set -euo pipefail
npm run check
```

Then make it executable:
```bash
chmod +x /Users/alexrojas/code/liveloop/main/.dev/checks.sh
```

This delegates to the single source of truth in `package.json`. Do not inline `tsc` or `node --test` invocations.

**Verify after creating:**
```bash
git add .dev/checks.sh
git ls-files .dev/   # must list .dev/checks.sh
```

### 3. Edit `.github/workflows/ci.yml`

Make exactly two edits to ci.yml:

**Edit A** — remove the stale `better-sqlite3` comment. Replace lines 32-34:
```yaml

      # python/make/g++ toolchain node-gyp needs, so no extra setup.
```

with:
```yaml

```

**Edit B** — replace the check step. Replace line 36:

Leave everything else unchanged: `on:` triggers, `concurrency:`, `strategy.matrix.node: [22, 24]`, `fail-fast: false`, `checkout@v5`, `setup-node@v5` with `cache: npm`, and `npm ci`.

## Edge cases

- **Do not** `git add -A` or `git add .` — this would accidentally stage local `.dev/plans/` or `.dev/workflows/` if the negation is misapplied.
- If `git ls-files .dev/` shows more than just `checks.sh`, the gitignore fix is wrong — stop and re-examine before committing.
- The file must be committed, not just on disk. The delivery checker and `actions/checkout` only see committed files.
- `set -euo pipefail` ensures any failure in `npm run check` exits nonzero; this is required for the checker to distinguish pass from fail.
